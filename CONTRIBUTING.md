# Contributing To FieldProof

Thanks for helping build public-interest audit infrastructure.

## Good First Contributions

- Improve the clickable prototype.
- Turn prototype screens into real React components.
- Draft audit packet schemas.
- Research field-device anti-spoofing options.
- Design offline map package workflows.
- Write user interview scripts for auditors, landowners, brands, and validators.
- Improve documentation and diagrams.

## Contribution Principles

- Optimize for field reality over technical novelty.
- Keep the core system self-hostable.
- Prefer open standards and portable data.
- Treat landowners and auditors as first-class users.
- Be careful with claims about verification. Say what the system proves, what it suggests, and what still requires human judgment.

## Suggested Workflow

1. Open or comment on an issue before large changes.
2. Keep pull requests focused.
3. Include screenshots for UI changes.
4. Include schema examples for data model changes.
5. Update docs when behavior or architecture changes.

## Project Areas

- `prototype`: clickable product flows and UX validation.
- `mobile`: offline-first field capture.
- `contracts`: on-chain audit registry, bounty escrow, appeals, and reputation.
- `storage`: decentralized evidence packet storage.
- `dashboard`: public farm, brand, and consumer views.
- `governance`: contribution rules, validator policy, and dispute processes.

## Code Of Conduct

Use the project to make the work more trustworthy, fair, and useful. Be direct, generous, and precise. Do not harass contributors, farmers, auditors, validators, or communities represented in audit data.
