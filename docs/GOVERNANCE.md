# Governance

FieldProof should be governed like public infrastructure, not a closed SaaS product.

## Starting Model

Recommended early governance:

- Maintainer team for code review and releases.
- Public roadmap and issue tracker.
- Transparent validator policy.
- Written conflict-of-interest rules.
- Public security reporting process.

## Decision Areas

Some decisions should require broader discussion:

- License changes.
- Token launch.
- Validator admission rules.
- Slashing or penalty rules.
- Data removal or redaction policy.
- Partnerships that could bias audit outcomes.

## Validator Governance

Validators should be selected for competence and independence.

Potential validator categories:

- Agricultural scientists.
- Local agronomists.
- NGO field experts.
- Cooperative representatives.
- Remote sensing analysts.

Validator decisions should be recorded with rationale and confidence, not just a binary vote.

## Landowner Rights

The project should preserve:

- Right to see audit records.
- Right to add context.
- Right to appeal adverse findings.
- Right to submit counter-evidence.
- Right to request correction of plainly wrong metadata.

## Data Ethics

Audit data can affect livelihoods. The system should avoid publishing unnecessary personal information, precise private details unrelated to the environmental claim, or data that exposes field workers to retaliation.
